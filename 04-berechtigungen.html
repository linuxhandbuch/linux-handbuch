KAP. 04

Berechtigungen

Linux ist ein Mehrbenutzersystem, bei dem jede Datei genau festgelegte Zugriffsrechte besitzt. Dieses Kapitel erklärt, wie man diese Rechte liest, ändert und versteht, wer welche Aktionen ausführen darf.

chmod 01

Ändert die Zugriffsrechte (Berechtigungen) einer Datei oder eines Ordners.

SYNTAX
$ chmod [rechte] [datei]
FLAGS
FlagBedeutung
u+xFügt dem Besitzer (user) Ausführrechte hinzu
g-wEntfernt der Gruppe Schreibrechte
755Setzt Rechte numerisch: rwx für Besitzer, r-x für Gruppe und Andere
BEISPIEL
$ chmod 755 skript.sh
OUTPUT
Kein sichtbarer Output. ls -l zeigt danach: -rwxr-xr-x skript.sh
HINWEIS

Die Zahlenschreibweise (z.B. 755) ist eine Summe: Lesen = 4, Schreiben = 2, Ausführen = 1. 7 bedeutet also alle drei Rechte.

chown 02

Ändert den Besitzer einer Datei oder eines Ordners.

SYNTAX
$ chown [benutzer] [datei]
FLAGS
FlagBedeutung
-RÄndert den Besitzer rekursiv für alle Unterordner und Dateien
[benutzer]:[gruppe]Ändert Besitzer und Gruppe gleichzeitig
BEISPIEL
$ sudo chown elias:elias bericht.txt
OUTPUT
Kein sichtbarer Output. ls -l zeigt danach elias als neuen Besitzer und neue Gruppe an.
HINWEIS

chown benötigt normalerweise sudo-Rechte, da man sonst nur Dateien ändern darf, die einem selbst gehören.

chgrp 03

Ändert nur die Gruppenzugehörigkeit einer Datei, ohne den Besitzer zu ändern.

SYNTAX
$ chgrp [gruppe] [datei]
FLAGS
FlagBedeutung
-RÄndert die Gruppe rekursiv für alle Unterordner
BEISPIEL
$ chgrp team bericht.txt
OUTPUT
Kein sichtbarer Output. Die Gruppe der Datei wird auf team gesetzt.
HINWEIS

chgrp ist im Prinzip eine Kurzform von chown, wenn man nur die Gruppe und nicht den Besitzer ändern möchte.

umask 04

Legt fest, welche Standardrechte neu erstellte Dateien und Ordner automatisch erhalten.

SYNTAX
$ umask [wert]
BEISPIEL
$ umask 022
OUTPUT
Kein sichtbarer Output. Neu erstellte Dateien erhalten danach standardmässig die Rechte 644, neue Ordner 755.
HINWEIS

Der umask-Wert wird von den maximalen Rechten (666 für Dateien, 777 für Ordner) abgezogen. umask 022 ist auf den meisten Systemen der Standard.

sudo 05

Führt einen Befehl mit Administratorrechten aus (superuser do).

SYNTAX
$ sudo [befehl]
FLAGS
FlagBedeutung
-iStartet eine vollständige Root-Shell
BEISPIEL
$ sudo apt update
OUTPUT
[sudo] Passwort für elias: ****
Fragt nach dem eigenen Benutzerpasswort und führt den Befehl danach mit erweiterten Rechten aus.
HINWEIS

sudo sollte man nur verwenden, wenn es wirklich nötig ist. Ein falscher Befehl mit sudo kann grösseren Schaden anrichten als ohne.

su 06

Wechselt zu einem anderen Benutzerkonto (switch user), z.B. direkt zu root.

SYNTAX
$ su [benutzername]
FLAGS
FlagBedeutung
-Lädt zusätzlich die Umgebung des Zielbenutzers vollständig
BEISPIEL
$ su -
OUTPUT
Passwort: ****
Wechselt danach vollständig ins root-Konto, erkennbar am # statt $ im Prompt.
HINWEIS

Auf modernen Ubuntu-Systemen ist das direkte root-Passwort meist nicht gesetzt. Dort wird stattdessen sudo bevorzugt.

whoami 07

Zeigt den Namen des aktuell angemeldeten Benutzers an.

SYNTAX
$ whoami
BEISPIEL
$ whoami
OUTPUT
elias
HINWEIS

whoami ist praktisch, um schnell zu prüfen, mit welchem Benutzerkonto man gerade arbeitet, besonders nach su oder sudo -i.

id 08

Zeigt die Benutzer-ID, Gruppen-ID und alle Gruppenmitgliedschaften des aktuellen Benutzers an.

SYNTAX
$ id [benutzername]
BEISPIEL
$ id
OUTPUT
uid=1000(elias) gid=1000(elias) gruppen=1000(elias),27(sudo),999(docker)
HINWEIS

An der Gruppe sudo in der Ausgabe erkennt man, ob ein Benutzer Administratorrechte über sudo besitzt.

groups 09

Zeigt an, in welchen Gruppen der aktuelle oder ein anderer Benutzer Mitglied ist.

SYNTAX
$ groups [benutzername]
BEISPIEL
$ groups elias
OUTPUT
elias : elias sudo docker
HINWEIS

groups zeigt eine übersichtlichere, kürzere Liste als id - ideal, wenn man nur die Gruppenzugehörigkeit sehen möchte.
