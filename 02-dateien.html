KAP. 02

Dateien & Verzeichnisse

Dieses Kapitel behandelt das Erstellen, Kopieren, Verschieben und Löschen von Dateien und Ordnern. Man lernt ausserdem, wie man Dateiinformationen abfragt und grosse Datenmengen effizient kopiert.

mkdir 01

Erstellt einen neuen Ordner.

SYNTAX
$ mkdir [optionen] [ordnername]
FLAGS
FlagBedeutung
-pErstellt bei Bedarf auch fehlende übergeordnete Ordner
BEISPIEL
$ mkdir -p projekte/2025/schule
OUTPUT
Kein sichtbarer Output. Der Ordnerpfad wird angelegt, auch wenn projekte/2025 vorher noch nicht existiert.
HINWEIS

Ohne -p schlägt der Befehl fehl, wenn ein übergeordneter Ordner noch nicht existiert.

touch 02

Erstellt eine leere Datei oder aktualisiert das Änderungsdatum einer bestehenden Datei.

SYNTAX
$ touch [dateiname]
FLAGS
FlagBedeutung
-t [zeit]Setzt ein bestimmtes Datum statt der aktuellen Zeit
BEISPIEL
$ touch notizen.txt
OUTPUT
Kein sichtbarer Output. Die Datei notizen.txt wird leer erstellt.
HINWEIS

touch wird oft benutzt, um schnell Platzhalter-Dateien anzulegen, z.B. beim Aufbau einer Ordnerstruktur.

cp 03

Kopiert Dateien oder Ordner.

SYNTAX
$ cp [optionen] [quelle] [ziel]
FLAGS
FlagBedeutung
-rKopiert Ordner rekursiv inklusive Inhalt
-vZeigt jede kopierte Datei im Terminal an
-iFragt vor dem Überschreiben nach
BEISPIEL
$ cp -r dokumente/ backup/
OUTPUT
Kein sichtbarer Output ohne -v. Mit -v wird jede kopierte Datei einzeln aufgelistet.
HINWEIS

Ordner können nur mit -r kopiert werden. Ohne diese Option gibt cp eine Fehlermeldung aus.

mv 04

Verschiebt Dateien oder Ordner, oder benennt sie um.

SYNTAX
$ mv [quelle] [ziel]
FLAGS
FlagBedeutung
-iFragt vor dem Überschreiben nach
-vZeigt den Vorgang im Terminal an
BEISPIEL
$ mv bericht_alt.txt archiv/bericht_2024.txt
OUTPUT
Kein sichtbarer Output. Die Datei wird verschoben und gleichzeitig umbenannt.
HINWEIS

mv ins gleiche Verzeichnis mit neuem Namen ist der Standardweg, um eine Datei umzubenennen.

rm 05

Löscht Dateien oder Ordner unwiderruflich.

SYNTAX
$ rm [optionen] [datei/ordner]
FLAGS
FlagBedeutung
-rLöscht Ordner rekursiv inklusive Inhalt
-fLöscht ohne Rückfrage, auch geschützte Dateien
-iFragt vor jedem Löschvorgang nach
BEISPIEL
$ rm -r alte_projekte/
OUTPUT
Kein sichtbarer Output. Der Ordner alte_projekte/ wird vollständig gelöscht.
HINWEIS

Es gibt keinen Papierkorb in der Kommandozeile. Man sollte rm -rf besonders vorsichtig verwenden, da gelöschte Dateien nicht wiederhergestellt werden können.

ln 06

Erstellt Verknüpfungen (Links) zwischen Dateien.

SYNTAX
$ ln [optionen] [ziel] [linkname]
FLAGS
FlagBedeutung
-sErstellt einen symbolischen Link (Verweis auf den Pfad)
BEISPIEL
$ ln -s /home/elias/dokumente/bericht.txt schnellzugriff.txt
OUTPUT
Kein sichtbarer Output. schnellzugriff.txt verweist danach auf die Originaldatei.
HINWEIS

Symbolische Links (-s) sind wie Verknüpfungen unter Windows. Ohne -s erstellt man einen harten Link, der sich anders verhält und seltener gebraucht wird.

rsync 07

Kopiert oder synchronisiert grosse Datenmengen effizient, da nur Änderungen übertragen werden.

SYNTAX
$ rsync [optionen] [quelle] [ziel]
FLAGS
FlagBedeutung
-aArchivmodus: erhält Rechte, Zeitstempel und Unterordner
-vZeigt den Fortschritt im Terminal an
--deleteLöscht im Ziel Dateien, die in der Quelle nicht mehr vorhanden sind
BEISPIEL
$ rsync -av dokumente/ backup/dokumente/
OUTPUT
sending incremental file list
bericht.txt
präsentation.pptx

sent 3.412.099 bytes  received 42 bytes
HINWEIS

Anders als cp überträgt rsync bei erneutem Ausführen nur die Dateien, die sich seit dem letzten Mal geändert haben. Das macht es ideal für Backups.

stat 08

Zeigt detaillierte Informationen zu einer Datei an, z.B. Grösse, Rechte und Zeitstempel.

SYNTAX
$ stat [datei]
BEISPIEL
$ stat bericht.txt
OUTPUT
File: bericht.txt
Size: 12288       Blocks: 24
Access: 2025-07-01 09:12:03
Modify: 2025-06-28 14:02:11
HINWEIS

stat zeigt deutlich mehr Details als ls -l, z.B. auch den letzten Zugriffszeitpunkt.

file 09

Zeigt den tatsächlichen Dateityp an, unabhängig von der Dateiendung.

SYNTAX
$ file [datei]
BEISPIEL
$ file unbekannt.dat
OUTPUT
unbekannt.dat: PDF document, version 1.7
HINWEIS

file ist praktisch, wenn eine Datei eine falsche oder fehlende Dateiendung hat und man wissen will, was wirklich drinsteckt.

rename 10

Benennt mehrere Dateien gleichzeitig anhand eines Musters um.

SYNTAX
$ rename [muster] [datei(en)]
BEISPIEL
$ rename 's/.jpeg/.jpg/' *.jpeg
OUTPUT
Kein sichtbarer Output. Alle .jpeg-Dateien im Ordner werden zu .jpg umbenannt.
HINWEIS

rename ist nicht auf jedem System vorinstalliert und nutzt reguläre Ausdrücke. Für einzelne Dateien reicht meist mv.
