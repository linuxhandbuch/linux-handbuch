KAP. 03

Dateiinhalt lesen

In diesem Kapitel lernt man, wie man den Inhalt von Textdateien anzeigt, durchsucht und weiterverarbeitet, ohne einen Editor öffnen zu müssen. Diese Befehle werden oft miteinander kombiniert (mit Pipes), um schnell Informationen aus Dateien herauszufiltern.

cat 01

Gibt den gesamten Inhalt einer Datei im Terminal aus.

SYNTAX
$ cat [datei]
FLAGS
FlagBedeutung
-nZeigt Zeilennummern an
BEISPIEL
$ cat notizen.txt
OUTPUT
Einkaufsliste:
- Milch
- Brot
- Kaffee
HINWEIS

cat eignet sich gut für kurze Dateien. Bei langen Dateien scrollt der Inhalt schnell aus dem sichtbaren Bereich - dafür eignet sich less besser.

less 02

Zeigt den Inhalt einer Datei seitenweise an, ohne alles auf einmal auszugeben.

SYNTAX
$ less [datei]
FLAGS
FlagBedeutung
/[suchbegriff]Sucht innerhalb von less nach einem Begriff (im Programm eingeben)
BEISPIEL
$ less logfile.txt
OUTPUT
Öffnet die Datei interaktiv. Man navigiert mit den Pfeiltasten, mit q beendet man less wieder.
HINWEIS

less lädt die Datei nicht komplett in den Speicher, weshalb es sich auch für sehr grosse Dateien eignet.

more 03

Ähnlich wie less, zeigt Inhalte seitenweise an, ist aber einfacher und älter.

SYNTAX
$ more [datei]
BEISPIEL
$ more logfile.txt
OUTPUT
Zeigt die erste Bildschirmseite. Mit der Leertaste blättert man weiter.
HINWEIS

less kann fast alles, was more kann, zusätzlich aber auch rückwärts blättern. In der Praxis wird meist less bevorzugt.

head 04

Zeigt die ersten Zeilen einer Datei an.

SYNTAX
$ head [optionen] [datei]
FLAGS
FlagBedeutung
-n [zahl]Legt die Anzahl der angezeigten Zeilen fest (Standard: 10)
BEISPIEL
$ head -n 5 logfile.txt
OUTPUT
09:12:03 Server gestartet
09:12:05 Verbindung hergestellt
09:13:41 Nutzer eingeloggt
09:14:02 Anfrage empfangen
09:14:03 Antwort gesendet
HINWEIS

head eignet sich gut, um schnell einen Überblick über eine Datei zu bekommen, ohne sie komplett zu öffnen.

tail 05

Zeigt die letzten Zeilen einer Datei an.

SYNTAX
$ tail [optionen] [datei]
FLAGS
FlagBedeutung
-n [zahl]Legt die Anzahl der angezeigten Zeilen fest
-fVerfolgt die Datei live und zeigt neue Zeilen sofort an
BEISPIEL
$ tail -f logfile.txt
OUTPUT
Zeigt die letzten 10 Zeilen und bleibt danach aktiv - neue Einträge erscheinen laufend, bis man mit Strg+C abbricht.
HINWEIS

tail -f wird sehr häufig verwendet, um Logdateien in Echtzeit zu beobachten, z.B. bei laufenden Servern.

grep 06

Durchsucht Dateien nach Textmustern und zeigt passende Zeilen an.

SYNTAX
$ grep [optionen] [muster] [datei]
FLAGS
FlagBedeutung
-iIgnoriert Gross-/Kleinschreibung
-rDurchsucht Ordner rekursiv
-nZeigt die Zeilennummer der Treffer an
-vZeigt nur Zeilen, die NICHT auf das Muster passen
BEISPIEL
$ grep -n "Fehler" logfile.txt
OUTPUT
42: 09:41:12 Fehler beim Verbindungsaufbau
108: 10:03:55 Fehler: Timeout
HINWEIS

grep gehört zu den am häufigsten genutzten Befehlen überhaupt und wird oft mit Pipes kombiniert, z.B. cat logfile.txt | grep Fehler.

wc 07

Zählt Zeilen, Wörter und Zeichen einer Datei (word count).

SYNTAX
$ wc [optionen] [datei]
FLAGS
FlagBedeutung
-lZählt nur Zeilen
-wZählt nur Wörter
-cZählt nur Zeichen (Bytes)
BEISPIEL
$ wc -l logfile.txt
OUTPUT
342 logfile.txt
HINWEIS

wc -l wird oft mit grep kombiniert, um die Anzahl der Treffer zu zählen, z.B. grep Fehler logfile.txt | wc -l.

sort 08

Sortiert die Zeilen einer Datei alphabetisch oder numerisch.

SYNTAX
$ sort [optionen] [datei]
FLAGS
FlagBedeutung
-nSortiert numerisch statt alphabetisch
-rKehrt die Sortierreihenfolge um
-uEntfernt doppelte Zeilen beim Sortieren
BEISPIEL
$ sort namen.txt
OUTPUT
Anna
Bernd
Clara
HINWEIS

Ohne -n sortiert sort Zahlen wie Text, wodurch z.B. 10 vor 2 stehen würde. Bei Zahlenkolonnen ist -n also wichtig.

uniq 09

Entfernt aufeinanderfolgende doppelte Zeilen aus einer Datei.

SYNTAX
$ uniq [optionen] [datei]
FLAGS
FlagBedeutung
-cZeigt zusätzlich, wie oft jede Zeile vorkam
BEISPIEL
$ sort namen.txt | uniq -c
OUTPUT
2 Anna
1 Bernd
3 Clara
HINWEIS

uniq erkennt nur direkt aufeinanderfolgende Duplikate. Deshalb wird die Datei meist vorher mit sort sortiert.

diff 10

Vergleicht zwei Dateien und zeigt die Unterschiede an.

SYNTAX
$ diff [datei1] [datei2]
FLAGS
FlagBedeutung
-uZeigt die Unterschiede im übersichtlichen unified-Format
BEISPIEL
$ diff -u bericht_alt.txt bericht_neu.txt
OUTPUT
-Der Umsatz stieg um 5%.
+Der Umsatz stieg um 8%.
HINWEIS

Zeilen mit - stammen aus der ersten, Zeilen mit + aus der zweiten Datei. So sieht man auf einen Blick, was sich geändert hat.

cut 11

Schneidet bestimmte Spalten oder Zeichenbereiche aus jeder Zeile einer Datei heraus.

SYNTAX
$ cut [optionen] [datei]
FLAGS
FlagBedeutung
-d [zeichen]Legt das Trennzeichen fest, z.B. Komma
-f [zahl]Wählt die Spaltennummer aus
BEISPIEL
$ cut -d',' -f1 kontakte.csv
OUTPUT
Anna
Bernd
Clara
HINWEIS

cut eignet sich gut für einfache, sauber strukturierte Dateien wie CSV. Bei komplexeren Fällen ist awk flexibler.

awk 12

Ein mächtiges Werkzeug zur Textverarbeitung, das Zeilen in Spalten aufteilt und gezielt verarbeitet.

SYNTAX
$ awk '[programm]' [datei]
FLAGS
FlagBedeutung
'{print $1}'Gibt die erste Spalte jeder Zeile aus
-F ','Legt ein eigenes Trennzeichen fest
BEISPIEL
$ awk -F',' '{print $2}' kontakte.csv
OUTPUT
anna@mail.ch
bernd@mail.ch
clara@mail.ch
HINWEIS

awk kann sehr komplex werden. Für den Einstieg reicht es, einzelne Spalten mit print $spaltennummer auszugeben.

sed 13

Sucht und ersetzt Text in Dateien automatisch, ohne einen Editor zu öffnen (stream editor).

SYNTAX
$ sed 's/[alt]/[neu]/g' [datei]
FLAGS
FlagBedeutung
-iÄndert die Datei direkt statt nur die Ausgabe anzuzeigen
BEISPIEL
$ sed 's/Fehler/Warnung/g' logfile.txt
OUTPUT
Zeigt logfile.txt mit allen Vorkommen von "Fehler" durch "Warnung" ersetzt, ohne die Originaldatei zu ändern.
HINWEIS

Ohne -i verändert sed die Originaldatei nicht, sondern gibt das Ergebnis nur im Terminal aus. Das ist praktisch zum sicheren Testen.
