KAP. 06

Hilfesystem

Man muss sich nicht alle Befehle auswendig merken. Linux bringt eingebaute Hilfesysteme mit, mit denen man jederzeit nachschlagen kann, wie ein Befehl funktioniert und welche Optionen er unterstützt.

man 01

Öffnet das Handbuch (manual) zu einem Befehl mit ausführlicher Beschreibung aller Optionen.

SYNTAX
$ man [befehl]
FLAGS
FlagBedeutung
/[suchbegriff]Sucht innerhalb der man-Seite nach einem Begriff (im Programm eingeben)
BEISPIEL
$ man ls
OUTPUT
Öffnet eine interaktive Textseite mit Beschreibung, Syntax und allen Optionen von ls. Mit q beendet man die Ansicht wieder.
HINWEIS

man ist die ausführlichste, aber auch technischste Hilfequelle. Für einen schnellen Überblick eignet sich --help oft besser.

--help 02

Zeigt eine kurze Übersicht der wichtigsten Optionen eines Befehls direkt im Terminal an.

SYNTAX
$ [befehl] --help
BEISPIEL
$ ls --help
OUTPUT
Usage: ls [OPTION]... [FILE]...
List information about the FILEs.

-a, --all      versteckte Dateien nicht auslassen
-l             lange Auflistung verwenden
HINWEIS

--help ist meist schneller als man, da man direkt im Terminal bleibt und keine neue Ansicht öffnet.

tldr 03

Zeigt vereinfachte Beispiele zu einem Befehl an, statt einer vollständigen technischen Beschreibung.

SYNTAX
$ tldr [befehl]
BEISPIEL
$ tldr tar
OUTPUT
tar
Archivierungswerkzeug, oft kombiniert mit Kompression.

- Archiv erstellen:
  tar cf archiv.tar ordner/

- Archiv mit gzip komprimieren:
  tar czf archiv.tar.gz ordner/
HINWEIS

tldr muss zuerst installiert werden (sudo apt install tldr). Für den schnellen Einstieg ist es oft verständlicher als man.
